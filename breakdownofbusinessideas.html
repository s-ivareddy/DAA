1. Transportation:

Dijkstra:

This algorithm is used to find the shortest path from a source node to all other nodes in a weighted graph.

  • Works with graphs that have non-negative edge weights.
  • Finds the shortest path from a single source to all other nodes.
  • Can be used with directed or undirected graphs.

In our business case, this algorithm can be used for:

  • Institutional Transportation: As the institutions need to travel to all the places and come back to the institution to pick up students. So, here Dijkstra's algorithm can be used to generate the shortest path to pick up students at the required stops.
  • Agricultural and Goods Transportation: In agricultural and goods transportation, they should visit multiple stores, fields, and godowns according to their requirements (sometimes multiple times in a single ride). So, here they can get the shortest path to visit all the places using this algorithm.
  • Local Transportation: They should drop the passengers at selected stops according to the passengers' preferences. So, they can use this algorithm to get the shortest path to drop all the passengers at selected stops.

Floyd-Warshall:

This algorithm is used to find the shortest path between all pairs of nodes in a weighted graph.

  • This is a dynamic programming algorithm.
  • It can handle both directed and undirected graphs.
  • Works even if the graph has negative edge weights, as long as there are no negative weight cycles.

In our business case, this algorithm can be used for:

  • Clinical Transportation: In clinical transportation, ambulances should go to the nearest hospital with the shortest possible distance. Floyd-Warshall can be used to get the shortest path.
  • Agricultural and Goods Transportation: In some cases, agricultural and goods transportation involves moving from one point to another selected point without stops. Floyd-Warshall can give them the shortest path to the destination.

Corporate and industrial transport needs both the Dijkstra and Floyd-Warshall because there may be intra-city and inter-city transportation according to their requirement as to get and supply goods or transportation for their employees. So, here they need both the algorithms.


Intra-City transportation always follows same path to connect all the places in the city and this path is generated by Dijkstra. They may change their path if there is some issue with road or with any external factors.


Long-Distance Transport(intra city transportation) this includes govt. busses, and trains if their service is between two places with no stops between they can use Floyd-Warshall's algorithm, or if there are stops in between they should use Dijkstra's algorithm, and here itself many of the times they will follow a fixed path which will be derived by Dijkstra or Floyd-Warshall according to the service.

2. Infrastructure:

  • Both Energy and Water should be supplied to each house from a single source(one source for a certain place street or area). So, here Dijkstra's algorithm can be used to get the least amount of pipes or wire required to connect each and every house.

  • Same as in waste management we need to collect waste and gather that at garbage which is located at outskirts of the city. So, there will be different trucks associated to collect garbage at different areas in city to gather the whole garbage at a single place. So, here we can use Dijkstra's algorithm to collet garbage at each area and to gather that garbage at city outskirts we can use Floyd-Warshall's algorithm and Dijkstra's algorithm as well

  • Community Infrastructure this includes construction of energy-efficient schools, clinics, and community centers designed with renewable energy, natural lighting, and green roofs. By this information we can create a graph(map) which includes each and every place(or location) which can be used by all the people in the city to know shortest path to travel.
